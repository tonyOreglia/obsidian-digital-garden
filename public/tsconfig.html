tsconfig

Root Properties

include ~ specifies which files to include

exclude ~ specifies which files to exclude

Compiler Options

These option should fall under the compilerOptions property within an object

lib ~ If program is running in the browser you want the dom type definitions.

allowJs ~ Allow JavaScript files to be imported inside your project, instead of just .ts and .tsx files.

skipLibCheck ~ skip type checking of declaration files (d.ts)

strict ~ stronger guarantees of correctness

noEmit ~ do not actually write the js output, this allows something like babel to do that. Then tsc is just a type checker really

esModuleInterop ~ By default, tsc treats CommonJS/AMD/UMD imports similar to ES6 modules. This is flawed in some cases. This fixes that.

module ~ sets the module system for the program. For example, esnext for the upcoming version of ecmascript

moduleResolution ~ Module resolution is the process the compiler uses to figure out what an import refers to. This specifies the module resolution strategy.

resolveJsonModule ~ enables importing JSON files

isolatedModules ~ this flag tells TypeScript to warn you if you write certain code that canâ€™t be correctly interpreted by a single-file transpilation process.

jsx ~ controls how jsx constructs are emmitted. For example, preserve emits .jsx files with JSX unchanged

incremental ~ Tells TypeScript to save information about the project graph from the last compilation to files stored on disk.
